# Отладка установки модуля Trust Me Sign

## Где искать логи ошибок

### 1. Лог установки модуля
**Путь:** `/bitrix/logs/trustme_sign_install.log`

Этот файл содержит подробную информацию о процессе установки:
- Какие шаги выполняются
- Какие файлы копируются
- Все ошибки и исключения

### 2. Лог ошибок PHP
**Путь:** `/bitrix/logs/php_errors.log`

Содержит все ошибки PHP, включая фатальные.

### 3. Логи веб-сервера
- Apache: обычно в `/var/log/apache2/error.log` или `/var/log/httpd/error_log`
- Nginx: обычно в `/var/log/nginx/error.log`

## Как просмотреть логи

### Через FTP/SFTP
1. Подключитесь к серверу
2. Перейдите в `/bitrix/logs/`
3. Откройте файл `trustme_sign_install.log`

### Через SSH
```bash
tail -f /path/to/bitrix/logs/trustme_sign_install.log
```

### Через админку Битрикс
Если есть доступ к файловой системе через админку, можно открыть файл напрямую.

## Что делать после просмотра лога

1. Найдите строки с "ОШИБКА" или "ERROR"
2. Скопируйте сообщение об ошибке
3. Проверьте:
   - Существуют ли все необходимые файлы
   - Правильные ли пути к файлам
   - Есть ли права на запись в `/local/activities/`

## Типичные ошибки

### "Directory not found"
- Проверьте, что модуль находится в `/local/modules/trustme.sign/` или `/bitrix/modules/trustme.sign/`
- Проверьте, что папка `activities/` существует в модуле

### "Permission denied"
- Установите права 755 на папку `/local/activities/`
- Убедитесь, что веб-сервер имеет права на запись

### "Class not found"
- Проверьте, что файл `include.php` модуля подключен
- Проверьте, что все классы определены правильно

## Временное включение отображения ошибок

Если нужно увидеть ошибку прямо на экране (только для отладки!), добавьте в начало `install/index.php`:

```php
ini_set('display_errors', 1);
error_reporting(E_ALL);
```

**⚠️ ВАЖНО:** Уберите это после отладки!
